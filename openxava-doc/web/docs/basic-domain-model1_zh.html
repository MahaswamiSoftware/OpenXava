引用多對一(ManyToOne) 作为描述列表（組合）

让我们从最简单的开始，先创建 Category 实体再将他与 Product 关联，将其作为一个组合显示。
Category 实体的代码是：
package com.yourcompany.invoicing.model;
 
import javax.persistence.*;
import javax.persistence.Entity;

import org.hibernate.annotations.*;
import org.openxava.annotations.*;

import lombok.*;
 
@Entity @Getter @Setter
public class Category {
 
    @Id
    @Hidden // 该属性不向用户显示。供内部识别
    @GeneratedValue(generator="system-uuid") // 通用唯一识别码 (1)
    @GenericGenerator(name="system-uuid", strategy = "uuid")
    @Column(length=32)
    String oid;
 
    @Column(length=50)
    String description;
 
}
他只有一个标识符和一个描述属性。这种情况我们使用 oid(1) 的算法生成 id。这生成器的优点是无需接触您的代码就能将应用程序迁移到另一个数据库（DB2、MySQL、Oracle、Informix 等）。其它 JPA 的 id 生成器都依赖数据库来生成id，从而使他们不像 UUID 一样可移植。

现在，我们将把 Product 与 Category 关联，在您的 Product 实体中添加 Category 及以下参照：
public class Product {
 
    ...
 
	@ManyToOne( // 该参照会在数据库作为一个关系保存
			fetch = FetchType.LAZY, // 该参照会在使用时才加载
			optional = true) // 该参照可以是空值（没有值）
	@DescriptionsList // 该参照会以组合的方式显示
	Category category; // 常规的 Java 参照
 
}

刻板印象

Product 实体至少需要价格等属性，另外有照片和备注的话最好。我们将使用刻板印象来做到这些，例如：您可使用字符串（String）来存储名称、备注或标识符也能使用 Big Decimal 来存储百分比、钱或数量。我们可将用同种类别在不同场景中使用。
理解刻板印象最好的方法就是看他如何使用。在 Product 实体中添加 price、photo 和 remark 属性，如下：
@Stereotype("MONEY") // 该属性用于存储货币
BigDecimal price; // BigDecimal通常用于钱的数量

@Stereotype("IMAGES_GALLERY") // 会有一个图片库
@Column(length = 32) // 该32长度的是保存图片库的键
String photos;

@Stereotype("MEMO") // 会有一个文本区域，该参照是用于大文本或相似
String remarks;
您已经了解如何使用刻板印象，现在您只需要使用刻板印象的名称，然后 OpenXava 会特殊处理它。现在运型应用并进入 Product 模块，在新建画面您可以看到：



如您所见，每个刻板印象在用户界面都会产生效果并有各自的大小、验证、编辑器等。此外还允许您很轻松的就能重用该功能，例如只需要将字符串属性标记为 @Stereotype(“IMAGES_GALLERY”)，您就可以使用图片库。
OpenXava 还有许多内置的刻板印象，例如：PHOTO、 LABEL、BOLD_LABEL、TIME、DATETIME、ZEROS_FILLED、HTML_TEXT、IMAGE_LABEL、EMAIL、 TELEPHONE、WEBURL、IP、ISBN、CREDIT_CARD、EMAIL_LIST、PASSWORD、MAC、FILE、FILES、 ICON、COORDINATES、等。
您也可以使用特定的注解替代刻板印象并获得相同的效果，例如：@Money 而不是 @Stereotype(“MONEY”)
// 注解替代刻板印象
      
@Money // 而不是 @Stereotype("MONEY") 
BigDecimal price; 
 
@Files // 而不是 @Stereotype("IMAGES_GALLERY") 
@Column(length=32)
String photos;
 
@TextArea // 而不是 @Stereotype("MEMO") 
String remarks;
一些可用的注解是：@Password、@Money、 @TextArea、@Label、@DateTime、@Discussion、@Icon、@Telephone、@IP、@EmailList、 @MAC、@StringTime、@HtmlText、@Coordinates、@Files ，@File、等。

可嵌入（Embeddable）

我们将在客户 Customer 添加地址 Address，由于地址并不跟其它客户共享，所以当客户被删除时，他的地址也一样会被删除。因此我们将地址概念建为可嵌入类，您可以在附录 B中了解更多信息。
接下来我们将 Address 添加至您的项目中，其代码如下：
package com.yourcompany.invoicing.model;

import javax.persistence.*;

import lombok.*;

@Embeddable // 我们使用 @Embeddable 而不是 @Entity
@Getter
@Setter
public class Address {

	@Column(length = 30) // 成员的参照与实体中的相同
	String street;

	@Column(length = 5)
	int zipCode;

	@Column(length = 20)
	String city;

	@Column(length = 30)
	String state;

}
现在您看到常规的类如何被注解为 @Embeddable，他的属性跟实体相同方式参照，尽管可嵌入类不支持实体的所有功能。
现在您能在任何实体中使用 Address，只需在 Customer 添加它的引用：
public class Customer {
 
    ...
 
	@Embedded // 这参照是指可嵌入类
	Address address; // 常规的 Java 参照
 
}
地址将跟客户存储在同一个表中，从用户界面视角，地址有一个框，如果您不喜欢可以使用 @NoFrame
// @NoFrame // 使用 @NoFrame 不显示地址（Address）的框
Address address; // 常规的 Java 参照
这是有和没框的差别
